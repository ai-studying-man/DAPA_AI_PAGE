# 방위사업청 AI 검색 랜딩 디자인 기준

## 목표

방위사업청 AI 검색의 첫 화면은 설명형 포털이 아니라 검색 진입 화면이다. 사용자는 방위사업청 홈페이지 정보, 공공데이터 파일, 오픈API를 한 번에 검색한다. 화면은 Google처럼 입력이 중심이지만, 방위사업청 고유의 공공기관 색감과 대한민국 방위력 이미지를 함께 전달한다.

## 시각 방향

- 첫 화면은 로고, 기관명, 검색창, 집중 검색 버튼만 남긴다.
- 배경은 무채색 빈 화면이 아니라 국방 획득과 K-방산을 상징하는 깊은 남색, 청색, 태극 적색, 금색 포인트를 사용한다.
- 스크롤 구간은 육군, 해군, 공군, 해병대 전력을 차례로 보여준다.
- 무기체계 비주얼은 실제 사진을 쓰기 전까지 SVG 기반의 선명한 실루엣 아트로 구성한다.
- 답변과 데이터 근거는 검색 후에만 노출한다.

## 로고

공식 홈페이지 MI 페이지에서 확인한 정부상징 기반 로고를 사용한다.

- 로컬 자산: `public/dapa-logo-combo.svg`
- 심볼 자산: `public/dapa-taeguk-symbol.svg`
- 화면에서는 로고 이미지와 큰 텍스트를 동시에 쓰지 않는다. 로고를 상단 식별자로 두고, H1은 검색 서비스의 주 진입점으로 사용한다.

## 화면 구조

1. Hero Search
   - 상단: 방위사업청 로고, 홈페이지 링크, 데이터셋·오픈API 링크
   - 중앙: `방위사업청` 타이틀과 검색창
   - 하단: 홈페이지 정보, 용어·규정, 군수품 조달정보, K-방산 국내, K-방산 국외 버튼

2. Scroll Visual Sections
   - 육군 전력: 지상 체계, 기동, 화력
   - 해군 전력: 함정, 해상 감시, 작전 지속성
   - 공군 전력: 항공기, 감시정찰, 정밀타격
   - 해병대 전력: 상륙, 기동, 합동작전

3. Result Layer
   - 검색 후에만 답변 패널 노출
   - 데이터 근거 요약은 `홈페이지`, `FILE`, `API` 카운트로 보여준다.

## 인터랙션

- 스크롤에 따라 각 군 전력 구간이 화면을 채운다.
- 검색창은 Hero 화면에서 가장 높은 시각 우선순위를 가진다.
- 집중 검색 버튼은 데이터 범위를 좁히는 역할을 한다.
- `활용 데이터 보기`는 별도 패널로 열어 사용 중인 홈페이지 정보, 파일데이터, 오픈API를 보여준다.

## 구현 메모

- React 상태는 검색어, 선택 목차, 답변, 출처 패널만 관리한다.
- 무기체계 비주얼은 외부 이미지 의존도를 낮추기 위해 SVG 컴포넌트로 구현한다.
- 실제 사진 자산을 확보하면 각 섹션의 SVG 배경을 `image-set()` 또는 `<picture>`로 교체할 수 있다.
