# 방위사업청 AI 공공데이터 에이전트 UI/UX 계획

작성일: 2026-05-20

## 목표

방위사업청 직원, 방산업체, 국민이 공개 데이터 속에서 필요한 방위사업 정보를 쉽게 찾고, K-방산의 국내 기반과 해외 확장을 데이터로 설명할 수 있는 질문형 AI 에이전트를 만든다.

초기 버전은 공공데이터와 LLM 모델을 직접 포함하지 않는다. 대신 홈페이지에 탑재 가능한 화면, 사용자 흐름, 목차 선택, 출처 표시, Ollama 연동 준비 구조를 구현한다.

## 참고한 화면 패턴

| 참고 사이트 | 관찰한 패턴 | 방사청 AI 에이전트 적용점 |
|---|---|---|
| 방위사업청 메인 | 큰 통합검색, 주요 서비스, 자주 찾는 메뉴 중심 | AI를 별도 포털이 아니라 통합검색을 보강하는 질문형 진입점으로 배치 |
| SAM.gov Contract Opportunities | 검색창, `Show active only`, 고급검색, 데이터 파일/API 연결 | 입찰공고·계약정보는 빠른 검색 버튼과 출처/데이터 연결을 같이 제공 |
| UK MOD Procurement at MOD | 목차형 안내, 신규 공급업체·SME 안내, DSP 등 외부 기회 연결 | 방산업체가 “처음 무엇을 해야 하는지” 버튼으로 안내 |
| Canada Defence procurement | 정부 공통 검색, 서비스와 정보 중심의 단순 페이지 | 국민용 설명은 행정 용어보다 쉬운 서비스명으로 노출 |
| Australia Defence Business & Industry | 산업 뉴스, 역량, 파트너십 중심 | K-방산 국내/해외 정보는 산업·기회·협력 관점으로 구성 |

## 최종 정보구조

1. 방위사업 용어·규격 검색
2. 법령·민원·자료 안내
3. 군수품 조달정보
   - 조달계획
   - 입찰공고
   - 입찰결과
   - 계약정보
4. K-방산 국내 정보
5. K-방산 해외 정보

`방위기술 R&D 동향`은 독립 목차에서 제외하고, 해당 데이터셋은 `K-방산 국내 정보`의 국내 기술개발 근거 데이터로 포함한다.

## 화면 구성 원칙

1. 홈페이지 히어로 안에 AI 질문창을 배치한다.
2. 주요 서비스 아래에는 자주 쓰는 AI 버튼을 6개 이하로 둔다.
3. 작업 화면은 `목차`, `채팅`, `근거와 키워드` 3열로 구성한다.
4. 데이터셋 원문 목록은 항상 좌측 또는 우측에 보이게 하되, 채팅보다 더 크게 만들지 않는다.
5. 답변은 파일데이터 기준과 오픈 API 최신 조회를 분리해서 표시한다.
6. 사용자의 질문과 답변은 로컬 저장소에 남기지 않는다.
7. 향후 Ollama 연동 시 `endpoint`, `model`, `messages`, `selectedDatasets`만 전달하도록 설계한다.

## 프로토타입 산출물

- `dapa_ai_agent_home_prototype.html`
- 정적 HTML/CSS/JS 단일 파일
- 데이터/API/LLM 미연결
- 목차 전환, 하위 항목 전환, 사용자 유형 전환, 추천 질문, 질문/답변 UI, 출처 데이터셋 표시 동작
- 외부 이미지·스크립트 의존 없음
- 로컬 저장소 미사용
- `활성 공고만`, `출처 보기`, `답변 복사`, `새 질문` 도구 포함

## 현재 구현된 사용자 흐름

1. 사용자는 메인 히어로의 AI 질문창 또는 주요 AI 버튼으로 진입한다.
2. 좌측에서 목차와 사용자 유형을 선택한다.
3. 군수품 조달정보는 조달계획, 입찰공고, 입찰결과, 계약정보로 세분화한다.
4. 중앙 채팅 영역은 질문과 답변을 보여주되 브라우저 저장소에는 남기지 않는다.
5. 우측 영역은 현재 선택 범위의 핵심 해시태그, 파일데이터 수, 오픈 API 수, 데이터셋 목록을 보여준다.
6. 답변 하단의 출처 표시는 토글 가능하며, 복사 버튼은 최신 AI 답변과 출처를 함께 복사한다.
7. 활성 공고만 버튼은 입찰공고 탐색 시 진행 중인 공고를 우선해야 한다는 조건을 명시한다.

## 향후 연동 포인트

```js
const OLLAMA_READY_CONFIG = {
  endpoint: "http://localhost:11434/api/chat",
  candidateModels: ["qwen2.5:7b", "qwen3:8b", "gemma3:12b"]
};
```

실제 연동 시에는 현재 선택된 목차, 하위 항목, 사용자 유형, 질문, 데이터셋 목록을 LLM 서버에 전달한다. 공공데이터 조회는 별도 데이터 어댑터에서 처리하고, 프론트엔드는 답변·출처·기준일만 렌더링한다.
